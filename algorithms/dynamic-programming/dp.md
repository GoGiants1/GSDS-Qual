# Dynammic Programming
: 큰 문제를 작은 문제로 나누어 풀기

## ‘Divide and conquer’과의 결정적인 차이점: **작은 문제가 중복되냐 아니냐**
    - Divide and conquer는 단지 큰 문제를 해결하기 어려워 작은 문제로 나누어 푸는 방법. 작은 문제에서 반복이 일어나지 않음
    - DP는 작은 부분문제들이 반복되고 그 답들이 바뀌지 않기 때문에 답을 ‘재사용’할 수 있음

## Memoization: 한 번 계산한 작은 문제를 저장해놓고 다시 사용하기 (보통 top-down에서 사용)

## 구현
    1. Bottom-up
    2. Top-down


**점화식을 잘 세우기** 
e.g) Fibonacci의 경우 '다음수열 = 이전 수열 + 두 단계 전 수열의' 이라는 식을 가짐
- F(5)를 구하기 위해 F(4)와 F(3)이 필요
- F(4)를 구하기 위해 F(3)과 F(2)가 필요
=> F(3)이 여러 번 필요 = 작은 문제가 반복되는 구조!
